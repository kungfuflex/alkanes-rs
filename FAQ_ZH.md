# Alkanes 常见问题

## Alkanes 是什么？

Alkanes 是一个将智能合约引入比特币 L1 的元协议。它将 WASM 二进制程序铭刻在交易的见证数据（witness data）中——与 Ordinals 铭刻图片和媒体文件所使用的空间完全相同。区别在于，Alkanes 铭刻的不是图片，而是可执行程序。

WASM（WebAssembly）是一种比 EVM 字节码更现代、更通用的程序格式。Rust、C、C++、AssemblyScript 等众多语言均可编译为 WASM，开发者无需学习 Solidity 这类专用语言，即可使用熟悉的工具链编写 Alkanes 智能合约。

## Alkanes 与 L2 有何不同？

传统的 L2 和 Rollup 依赖中心化排序器（sequencer），或由一组经许可的 PoA/PoS 验证者来批量打包交易并提交至 L1。这引入了信任假设、活性依赖以及比特币之外的额外基础设施。

Alkanes 没有这些问题。没有排序器，没有独立的验证者集合，也没有链下执行环境。每一次状态变更都在比特币 L1 上逐笔交易结算。该协议是比特币本身的轻量扩展——任何运行 Alkanes 索引器的节点都可以仅凭比特币区块链数据独立推导出完整状态。

## Alkanes 与 Ordinals 相比如何？

Ordinals 证明了比特币的见证空间可以承载任意数据——图片、文本、视频——并且社区愿意在此基础上构建和采用新协议。Alkanes 延续了这一思路，并将其应用于计算领域：铭刻的不再是一张 JPEG，而是一个可被调用、组合和执行的 WASM 二进制程序。

Ordinals 赋予比特币 NFT 和同质化代币（BRC-20）的能力，而 Alkanes 赋予比特币可编程智能合约与链上状态的能力。

## 为什么说 Alkanes 是"主权 Rollup"？

主权 Rollup 是指其规范状态完全由 L1 数据的确定性转换所定义。Alkanes 完全符合这一定义：

- **数据可用性是隐含的。** 所有交易数据都存储在比特币 L1 上，无需信任或支付额外的数据可用层（DA layer）。
- **状态是 L1 数据的纯函数。** 任何节点都可以通过 Alkanes 索引器重放比特币区块链，得到完全一致的状态。除比特币自身的共识机制外，不存在任何外部输入或额外共识。
- **逐笔交易结算。** 每一笔包含 Alkanes 操作的比特币交易，在被打包进区块的那一刻即完成结算。除比特币本身的确认模型外，没有批处理延迟或最终性窗口。

最终的效果是：一个继承了比特币安全性与去中心化特性的系统，同时具备可编程能力——而无需承担传统 L2 架构所带来的信任假设。
