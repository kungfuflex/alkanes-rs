## Q1: Alkanes（包括 FROST）如何保障比特币智能合约的安全性？

Alkanes 的安全性建立在多层防御机制之上：

**继承比特币 L1 安全性。** Alkanes 没有独立的共识机制、排序器或验证者集合。所有状态都是比特币链上数据的纯函数——任何人运行 Alkanes 索引器即可独立验证全部状态，无需信任任何第三方。

**WASM 沙箱执行与燃料计量。** 智能合约在确定性的 WASM 沙箱中运行，具备严格的内存隔离。每次执行都有燃料（fuel）计量机制，防止无限循环和拒绝服务攻击——合约消耗的计算资源超出限额即被终止。

**内存安全与输入校验。** 代码库包含专门的内存安全测试（`memory_security_tests.rs`），验证整数溢出保护（使用 u128 最大值等极端输入）和畸形数据处理。所有异常输入均被优雅拒绝，索引器不会因恶意构造的数据而崩溃。

**Subfrost（frBTC）采用比特币原生密码学。** frBTC 的包装与解包基于比特币原生的 Taproot/Schnorr 签名方案，不引入自定义密码学原语，安全性与比特币本身一致。

**UTXO 过滤保护。** 系统通过地址级 UTXO 过滤（`SpendablesByAddress`）防止用户意外花费包含铭文、符文或 Alkanes 资产的 UTXO。

## Q2: 这项技术能做什么？

Alkanes 赋予比特币 L1 完整的可编程能力：

**部署智能合约。** 开发者可以用 Rust、C、C++、AssemblyScript 等语言编写智能合约，编译为 WASM 后铭刻上链。合约可被调用、组合和执行，支持链上状态管理。

**可编程比特币（frBTC）。** 通过 Subfrost 协议，用户可将 BTC 包装为 frBTC。frBTC 是 Alkanes 生态中的可编程比特币，可参与 DeFi 操作，同时保持与原生 BTC 的 1:1 兑换关系。

**链上代币创建与 DeFi 基础设施。** 支持创建同质化代币，以及基于自动做市商（AMM）的流动性池。代码库已包含完整的池发现、储备查询、交换历史和 K 线数据等 API 端点。

**跨合约组合。** 合约之间可以相互调用，实现复杂的链上逻辑组合——这是构建 DeFi 协议栈的基础。

**浏览器钱包集成。** 终端用户可通过 UniSat、Xverse 等主流比特币钱包直接与 Alkanes 智能合约交互，无需使用命令行工具。

## Q3: 为什么全球其他项目需要我们的技术？

**唯一生产就绪的比特币 L1 主权 Rollup。** 目前市场上没有其他协议能在不引入排序器或额外共识层的前提下，在比特币 L1 上实现完整的智能合约功能。

**无排序器 = 无单点故障。** 传统 L2 依赖中心化排序器，一旦排序器宕机，整条链即停摆。Alkanes 没有这个问题——只要比特币网络运行，Alkanes 就运行。

**WASM 而非 EVM。** 支持 Rust、C、C++、AssemblyScript 等主流语言，开发者无需学习 Solidity。WASM 是 W3C 标准，拥有远比 EVM 更广泛的工具链和开发者生态。

**逐笔交易结算。** 每笔交易在被打包进比特币区块的那一刻即完成最终结算，没有批处理延迟或最终性窗口。

**隐含的数据可用性。** 所有数据存储在比特币 L1 上，无需支付或信任额外的数据可用层（DA layer），降低了成本和复杂度。

**面向全球金融的市场定位。** 我们的目标受众是广义上的金融行业。我们正在将 EVM 上已被验证的所有金融基础设施在比特币上重新构建和优化。比特币拥有远超其他区块链的市场规模，我们致力于让国际机构和零售用户都能无摩擦地参与这个全新的世界金融体系和基础架构。

**鼓励开发者共建生态。** Subfrost 对生态项目进行战略投资并帮助其成长。例如，我们的区块浏览器就是与社区开发者合作的项目；链上规范的 AMM 合约由 Flex 提供支持，现已成为比特币上类似 Uniswap 的通用交易系统。随着越来越多的比特币生态开发者发现 Alkanes 是可靠的、满足 AI 未来需求的解决方案，我们的生态将持续壮大。

**协作与包容。** Subfrost 始终定位为一家协作型、包容性的公司，不仅支持 Alkanes，还支持 BRC2.0、Arch，以及即将到来的 MIDL。与 ADA 的新合作伙伴关系将比特币和 frBTC 带入其他区块链，推动 Subfrost 成长为 Layer 0 级别的框架。
